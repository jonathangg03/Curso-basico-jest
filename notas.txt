Aplicaciones de testing para Async/Await

-Por convención, es mejor, sí vamos a usar el async await, no usar el .then, porque sería regundante, lo que 
    haremos será guardar la resolución de la promesa en una variable.

-Para probar el resultado en caso de error, vamos a hacer petición a la API http://httpstat.us/404 (aquí colocamos
    404, pero puede ser cualquier otro codigo). Ahorá, la petición a esta nos traerá un mensaje de error, por lo 
    que podremos probar el resultado del rechazo de esta petición.
        .El mensaje de error de esta api es: 'Request failed with status code 404' (o cualquier otro código).
        .Para este caso, no esperamos la resolición de la promesa, sino, esperamos el expect.

-En un test podemos tener varios expect, donde se prueben diferentes situaciones. Como ejemplo, probar que una 
    promesa se resuelva tanto con error, como con exito.
